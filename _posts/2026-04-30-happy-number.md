# 202. 快乐数

**题目链接：**[LeetCode 202. 快乐数](https://leetcode.cn/problems/happy-number/description/)

**题目描述：**

编写一个算法来判断一个数 `n` 是不是快乐数。

**「快乐数」** 定义为：

- 对于一个正整数，每一次将该数替换为它每个位置上的数字的平方和。
- 然后重复这个过程直到这个数变为 1，也可能是 **无限循环** 但始终变不到 1。
- 如果这个过程 **结果为** 1，那么这个数就是快乐数。

如果 `n` 是 _快乐数_ 就返回 `true` ；不是，则返回 `false` 。

**示例 1:**

	输入：n = 19
	输出：true
	解释：
	1^2 + 9^2 = 82
	8^2 + 2^2 = 68
	6^2 + 8^2 = 100
	1^2 + 0^2 + 0^2 = 1

**示例 2:**

	输入：n = 2
	输出：false

**提示**:

	1 <= n <= 2^31 - 1

## 双指针法

我认为这类题目的第一步还是找规律，从1开始分析一下这道题有没有循环：

| 1   | 2   | 3     | 4   | 5     | 6   | 7   |
| --- | --- | ----- | --- | ----- | --- | --- |
| 1   | 4   | 9     | 16  | 25    | 36  | 49  |
|     | 16  | 81    |     | 29    | 45  | 97  |
|     | 37  | 65    |     | 85=58 | 41  | 130 |
|     | 58  | 61=16 |     |       | 17  | 10  |
|     | 89  |       |     |       | 82  | 1   |
|     | 145 |       |     |       | 68  |     |
|     | 42  |       |     |       | 100 |     |
|     | 20  |       |     |       | 1   |     |
|     | 4   |       |     |       |     |     |
简单分析，所有的非快乐数都应当是会进入到一个循环，那么最简单的暴力+快慢指针方法即可判断一个数是否会进入循环，进入则说明是非快乐数。

**实现代码：**

```cpp

class Solution {
public:
    bool isHappy(int n) {
        int slow = n, fast = n;

        do {
            slow = getStepResult(slow);
            fast = getStepResult(getStepResult(fast));
        } while (slow != fast);

        return slow == 1;
    }

    int getStepResult(int n) {
        int result = 0;

        while (n > 0) {
            int bit = n % 10;
            result += bit * bit;
            n = n / 10;
        }

        return result;
    }
};

```

**执行结果：**
- 执行用时分布：0ms，击败100.00%
- 消耗内存分布：7.79MB，击败77.62%

**思路总结：**

没什么好说的，简单走一轮手搓看一下循环是不是有限的，毕竟题目已经告诉了我们非快乐数会进入循环。只要循环是可控的，就可以直接用快慢指针来判断。
