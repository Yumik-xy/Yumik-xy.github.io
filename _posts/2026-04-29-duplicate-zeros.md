# 1089. 复写零

**题目链接：**[LeetCode 1089. 复写零](https://leetcode.cn/problems/duplicate-zeros/description/)

**题目描述：**

给你一个长度固定的整数数组 `arr` ，请你将该数组中出现的每个零都复写一遍，并将其余的元素向右平移。

注意：请不要在超过该数组长度的位置写入元素。请对输入的数组 **就地** 进行上述修改，不要从函数返回任何东西。

**示例 1:**

	输入：arr = [1,0,2,3,0,4,5,0]
	输出：[1,0,0,2,3,0,0,4]
	解释：调用函数后，输入的数组将被修改为：[1,0,0,2,3,0,0,4]

**示例 2:**

	输入：arr = [1,2,3]
	输出：[1,2,3]
	解释：调用函数后，输入的数组将被修改为：[1,2,3]

**提示**:

	1 <= arr.length <= 104
	0 <= arr[i] <= 9

## 倒序法

在完成 [[2026-04-27-move-zeroes]] 后，该问题第一时间就会想到用双指针的方式来实现。

但一经思考就会发现，如果从左向右遍历：

```
[1,0,2,3,0,4,5,0] -> [1,0,0,2,0,4,5,0]
```

就会导致后侧的数据被覆盖，这样排序就没法直接在原数组里进行，必须要依赖外部数组。

很显然的，必须要从左向右对数据进行遍历。如果反向遍历，我们就需要定位这个元素所需要放置的位置。其等于该点前序的0的个数，加上当前的位置坐标：

$$ Index_{new}(n)=CountOfZero(n)+Index_{cur}(n) $$

因此可以考虑做一个等长的桶，记录每一个数字新的坐标，然后对原始数组重新赋值：

```
[1,0,2,3,0,4,5,0] -遍历前序0的格式-> [0,0,1,1,1,2,2,2] -计算得到新下标-> [0,-,3,4,-,7,8,-]

得到：[1,0,0,2,3,0,0,4] 
```

这时候我们发现，其对前序0的统计是顺序的，如果我们反过来看这个前序0的数组。

先对0进行数据统计得到3，反向进行遍历计算数字的新坐标：
- 非0数字，计算后直接填充到新坐标
- 0数字，对统计的0进行减1，并填充到新坐标和新坐标+1的位置：

```
zero_index = 3
length = 8

---------------------------------------
fast_index = 7
[1,0,2,3,0,4,5,0]
               ↑ 指向0时，zero_index = 3 - 1 = 2，计算该点的坐标 7 + 2 = 9 > 8 丢弃

--------------------------------------- ↓ ... 
fast_index = 5
[1,0,2,3,0,4,5,0]
           ↑ 计算该点的坐标 index = 5 + 2 = 7 < 8
           写入到 index 位置
[1,0,2,3,0,4,5,4]

---------------------------------------
fast_index = 4
[1,0,2,3,0,4,5,0]
         ↑ 指向0时，zero_index = 2 - 1 = 1，计算该点的坐标 index = 4 + 1 = 5 < 8 得到如下
         写入到 index 和 index + 1 位置
[1,0,2,3,0,0,0,4]

--------------------------------------- ↓ ... 

最后完成遍历
```

**实现代码：**

```C++

class Solution {
public:
	void duplicateZeros(vector<int>& arr) {
		int n = arr.size();

		int zero_index = 0;
		for (int num : arr) {
			if (num == 0)
				zero_index++;
		}

		for (int fast = n - 1; fast >= 0; fast--) {

			if (arr[fast] != 0) {
				int new_pos = fast + zero_index;
				if (new_pos < n) {
					arr[new_pos] = arr[fast];
				}
				
			} else {
				zero_index--;
				int new_pos = fast + zero_index;
				
				if (new_pos < n) {
					arr[new_pos] = 0;
				}
				if (new_pos + 1 < n) {
					arr[new_pos + 1] = 0;
				}
			}
		}
	}
};

```

**思路总结：**

在此方案下，整个工作的时间复杂度只有`O(n)`。

我们的思路还是先从从左到右遍历位移的方式来更新数据，但发现在题目的要求在原数组中进行位移，在补0后要保证后续的数字没有覆盖，必须倒序对所有数字向后进行位移。

因此顺序遍历肯定是不合适的，那么就需要考虑逆序遍历的方式，通过计算数字所在的坐标来对数据进行位移。这样思路就很明了了，画图计算并优化即可。
