# 283. 移动零

**题目链接：**[LeetCode 283. 移动零](https://leetcode.cn/problems/move-zeroes/description/)

**题目描述：**

给定一个数组 `nums`，编写一个函数将所有 `0` 移动到数组的末尾，同时保持非零元素的相对顺序。

**请注意** ，必须在不复制数组的情况下原地对数组进行操作。

**示例 1:**

	输入: nums = [0,1,0,3,12]
	输出: [1,3,12,0,0]

**示例 2:**

	输入: nums = `[0]`
	输出: [0]

**提示**:

	 1 <= nums.length <= 104
	-231 <= nums[i] <= 231 - 1

**进阶：**你能尽量减少完成的操作次数吗？

## 冒泡排序

首先想到最基础的冒泡，即遍历相邻的两个数字，如果前继继数字是0后继不是0，则交换这两个数字。

**实现代码：**
```C++

class Solution {
public:
	void moveZeroes(vector<int>& nums) {
		int n = nums.size();
		for (int i = 0; i < n; i++) {
            for (int j = 0; j < n - 1; j++) {
                if (nums[j] == 0 && nums[j + 1] != 0) {
                    swap(nums[j], nums[j + 1]);
                }
            }
        }
	}
};

```

**执行结果：**
- 执行用时分布：998ms，击败5.01%
- 消耗内存分布：23.31MB，击败72.56%

**思路总结：**

很显然，因为使用了冒泡排序，整个方案的时间复杂度达到了`O(n^2)`。

但我们从这次分析发现，因为0每次只往后移动一个位置，到底首位0需要移动接近N次才能移动到目标位置，如果可以一次性移动到最后，就只需要遍历一次数据即可。

## 双指针

如果使用双指针，一个从前序遍历寻找0，另一个从后续遍历寻找0可以插入的位置，是否就可以实现一次遍历解决问题了呢？

可是我们传入的是一个数组而非一个链表，当数字从列表中抽出时，后继数据无法自动向前补位，因此该方案还需要做一定的修正。

其实0不是一个具体数值，我们可以把它当做一个空格，如下示例：

```
[0, 1, 2, 0, 3] -> [ , 1, 2,  , 3] -> [1, 2, 3,  , ] 
```

很显然我们就可以发现，只需要顺序复制顺序到对应的位置，剩余空位置补零即可。

因此双指针的方式改变为快慢指针，快指针指向第一个非0的数字，而慢指针指向该数字需要插入的位置：

```
[0, 1, 2, 0, 3] -> [1, 1, 2, 0, 3] -> [1, 2, 2, 0, 3]  -> [1, 2, 3, 0, 3] 
 ↑  ↑                  ↑  ↑                  ↑     ↑
 S  F                  S  F                  S     F
```

当快指针指向了最后一个数字，说明排序已经完成，慢指针之后的所有数据均置为0即可。

**实现代码：**

```kotlin
class Solution {
public:
    void moveZeroes(vector<int>& nums) {
        int slow = 0;
        for (int fast = 0; fast < nums.size(); fast++) {
            if (nums[fast] != 0) {
                nums[slow] = nums[fast];
                slow++;
            }
        }
        for (int i = slow; i < nums.size(); i++) {
            nums[i] = 0;
        }
    }
};
```

**执行结果：**
- 执行用时分布：0ms，击败100.00%
- 消耗内存分布：23.31MB，击败70.93%

**思路总结：**

整体方案实际进行了n+n-N+1次遍历，因此整个方案的时间复杂度只有`O(n)`。

在这里的思路主要是从一个日常操作的双指针方向，发现其不可达后，发现0这是一个具体的占位数值。通过“视觉”上的调整，发现其只是一个简单的位移和补0操作，换用最基础的快慢指针方式即可实现。
